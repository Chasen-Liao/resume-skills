# JD 简历定制器 (JD Tailorer)

这个技能帮助用户根据具体职位描述（JD）来定制简历，提高简历与目标岗位的匹配度。

## 核心理念

- 不同公司和岗位关注点不同，一份通用简历不够
- 核心策略：**关键词匹配 + 内容重排序 + 术语对齐**
- 不编造经历，只优化呈现方式
- 沿用 resume-builder 生成的模板格式，保持视觉一致性
- ATS 友好：定制后的关键词策略天然提升 ATS 评分

## 前置条件

需要有基础简历文件。可以是：
- resume-builder 生成的 `resume.html`
- 用户提供已有的简历（粘贴文本或上传文件）

如果用户还没有基础简历，建议先用 resume-builder 创建一份。

## 工作流程

### 第一步：获取 JD

询问用户获取 JD 的方式：

- 粘贴 JD 文本（最常用）
- 上传 JD 文件（PDF/Word/图片）
- 提供职位链接（让用户粘贴链接内容）

### 第二步：JD 分析

解析 JD 内容，提取关键信息：

1. **岗位核心要求** — 必备技能、经验年限、学历要求
2. **加分项** — 优先考虑的附加条件
3. **关键词提取** — 技术栈名词、行业术语、软技能关键词
4. **岗位职责** — 该岗位的主要工作内容
5. **公司文化暗示** — JD 中体现的公司风格（创新驱动/稳定可靠/快节奏等）

将分析结果用简短的摘要呈现给用户，让用户确认理解是否准确。

### 第三步：简历匹配分析

对比基础简历和 JD 要求：

1. **匹配点**：简历中已有的、与 JD 高度相关的经验和技能
2. **缺失点**：JD 要求但简历中未体现的（如实告知用户，不编造）
3. **弱化点**：简历中有但与当前 JD 关联度低的内容（可以保留但降低优先级）

### 第四步：内容定制

基于分析结果，调整简历内容。定制过程遵循 resume-builder 的写作规范（`references/content-writing.md`）。

**关键词优化**：
- 将简历中的术语与 JD 统一（如「深度学习」→「LLM 大语言模型」如果 JD 这样写）
- 增加 JD 中频繁出现的关键词到技能标签中（仅限真实掌握的）
- 在项目描述中自然地融入 JD 相关术语
- ⚠️ 目标：核心 JD 关键词覆盖率 ≥ 70%，但切勿 keyword stuffing

**板块重排序**：
- 将 JD 最关注的经验/技能板块前置
- 弱化不相关的板块（保留但不突出）
- 应届生岗位：项目经验仍建议在校经历之前

**内容微调**：
- 重新措辞项目描述，突出与 JD 相关的成果
- 调整自我评价，对齐目标岗位的定位（三句话结构）
- 每条经历保持至少一个量化数字（STAR 法则）
- 不添加虚假经历；所有信息必须真实

**区分度说明**：每次定制后，对比定制版和通用版的差异点（改了哪些关键词/重排顺序），让用户清楚看到调整了什么。这比默默替换更好，也帮助用户理解为什么要定制。

### 第五步：生成定制文件

1. **创建子文件夹**：`tailored/<公司名>-<岗位>/`
2. **生成定制 HTML**：基于 resume-builder 模板格式，填充定制内容。沿用基础简历的 CSS 风格（从 `references/css/` 中对应风格文件读取配色/字体）
3. **保持 A4 布局**：同样支持 A4 纸打印和 PDF 导出
4. **强约束：必须单页**：定制版同样必须控制在 1 页 A4 以内。调整策略同 resume-builder（压缩间距/缩减字号/精简文字/双栏布局）
5. **ATS 验证**：定制后检查关键词是否覆盖了 JD 核心要求的 70% 以上

### 第五步半：PDF 导出验证（强制执行）

与 resume-builder 第三步半相同，定制版 HTML 生成后也必须导出 PDF 验证页数。页数 > 1 则必须修复。

**输出结构**：
```
resume/
├── resume.html                       # 通用版简历
├── tailored/
│   ├── alibaba-前端开发/
│   │   ├── resume.html               # 针对阿里的定制版
│   │   └── matching-analysis.md      # 匹配分析报告
│   └── tencent-后端开发/
│       ├── resume.html
│       └── matching-analysis.md
```

### 第六步：生成匹配分析报告

为每个定制版本生成一份简洁的分析报告 `matching-analysis.md`：

```markdown
# JD 匹配分析 — <公司名> <岗位>

## JD 关键词提取
- 核心技术：...
- 加分技能：...
- 软技能：...

## 匹配评估
- 整体匹配度：X/10（基于关键词覆盖率估算）
- 强匹配（已有且 JD 高频要求）：
- 需补充（JD 要求但候选人不具备）：...（如实标注）
- 弱化项（简历有但与 JD 关联低）：...

## 定制策略
- 关键词替换：X → Y（N 处）
- 板块重排：XX 前置，YY 后置
- 措辞调整：Z 个项目描述重新聚焦到 JD 方向
- 自我评价重写：对齐到 [岗位核心要求]

## 定制版 vs 通用版差异
| 方面 | 通用版 | 定制版 |
|------|--------|--------|
| 核心关键词 | ... | ... |
| 板块顺序 | ... | ... |
| 自我评价 | ... | ... |
```

## 重要约束

- **绝不编造经历**：如果 JD 要求某项技能而候选人确实不具备，告知用户但不强行添加
- **保持真实性**：所有修改仅限于措辞优化和呈现方式调整
- **沿用模板**：定制版 HTML 必须沿用基础简历的视觉风格和布局

## 示例对话

**用户**: 帮我针对腾讯的 JD 改一下简历
**Claude**: 好的，请把腾讯的 JD 内容粘贴给我，我会先分析一下这个岗位的核心要求。

**用户**: [粘贴 JD 文本]
**Claude**: 我已经分析了这个 JD，核心要求是：
- React/Vue 前端框架
- 3 年以上经验
- 有大型项目架构经验
...
我注意到你简历中的「电商后台管理系统」项目与这个岗位高度相关，我会把它放到更靠前的位置。另外 JD 中反复提到「性能优化」，你的项目中有提到吗？

## 与 resume-builder 的协作

两个技能形成完整工作流：
1. **resume-builder** → 建立通用基础简历
2. **jd-tailorer** → 针对不同 JD 生成定制版本

jd-tailorer 依赖 resume-builder 的模板格式，但也能处理用户已有的其他格式简历。
