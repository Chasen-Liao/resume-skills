# 简历构建器 (Resume Builder)

这个技能指导如何通过对话方式，为用户创建一份**视觉精美、排版专业**的 A4 纸 HTML 简历。

## 核心理念

- 简历是个人品牌的第一印象，需要独特且有记忆点
- 排版要干净利落，便于招聘官快速扫描
- 设计风格遵循内置的美学标准：大胆、有辨识度、拒绝通用审美。详见 `references/design-guidelines.md`
- **目标输出**：一个独立的 HTML 文件，内嵌 CSS/JS，A4 尺寸排版，可直接浏览器打印为 PDF

## 工作流程

### 第一步：信息收集（对话式面试）

与用户进行对话，收集以下信息板块。逐个板块提问，不要一次抛出所有问题。

**写作规范**：收集信息时参考 `references/content-writing.md`，把握 STAR 法则、数据量化、ATS 兼容等要点。用户提供模糊描述时主动追问量化数据。

**基础信息**：
- 姓名、联系方式（手机、邮箱、城市）
- 求职意向（目标岗位、期望城市、薪资范围可选）
- 个人网站/GitHub/LinkedIn（可选）

**教育背景**：
- 学校名称、专业、学历、起止时间
- GPA 或排名（可选）
- 相关课程或荣誉（可选）

**实习/工作经历**（核心板块）：
- 公司名称、岗位、起止时间
- 每个经历提取 2-4 个关键成果（用 STAR 法则：情境-任务-行动-结果）
- 量化数据优先（提升了 X%，完成了 Y 个项目，管理了 Z 人团队）
- ⚠️ 每一条必须有至少一个数字。用户说「参与」「负责」时追问：具体做了什么？成果可量化吗？

**项目经验**（核心板块）：
- 项目名称、角色、时间
- 项目简介（1-2 句话）
- 核心技术栈
- 个人贡献和成果（2-3 点，每条至少一个量化数字）
- ⚠️ 应届生建议将项目经历**前置到实习经历之前**，这是能力证明的关键

**技能标签**：
- 编程语言、框架、工具、平台
- 语言能力（中/英/其他）
- 可分级：精通/熟练/了解
- 控制在 8-12 个核心技能，不列无关技能
- ATS 兼容：使用语义化 `<ul>` 和 `<li>` 列表并通过 CSS inline 排列，而非独立色块标签

**校园经历**（应届生加分）：
- 社团活动、志愿者、竞赛获奖
- 担任角色和主要贡献

**自我评价**（可选）：
- **50 字以内**，严格三句话结构：
  1. 专业定位（"XX 专业应届生，主攻 XX 方向"）
  2. 核心能力（"具备 XX + XX + XX 能力"）
  3. 职业目标（"期望在 XX 领域深耕，为团队贡献 XX 价值"）
- 禁止空洞词汇：「吃苦耐劳」「性格开朗」「学习能力强」等无区分度的套话

### 第二步：风格确认

在收集完信息后，让用户确认简历风格。提供一个简洁的选择列表：

1. **现代简约** — 黑白为主，大量留白，几何线条，字体优雅
2. **经典商务** — 深蓝或深灰主调，稳重专业，传统排版
3. **创意个性** — 独特配色，不对称布局，展示个性（适合设计/创意岗位）
4. **日式极简** — 极简主义，柔和色彩，留白呼吸感强
5. **科技感** — 深色/渐变，几何元素，适合技术岗位

用户也可以描述自己想要的风格，由技能根据内置的美学指南进行创造。生成前，先阅读 `references/design-guidelines.md` 获取设计方向指导。

### 第三步：生成 HTML 简历

基于收集的信息和选择的风格，生成一个独立的 HTML 文件。

**技术约束**：
- 必须是一个独立的 HTML 文件（inline CSS/JS，无外部依赖）
- 页面尺寸严格为 A4（210mm × 297mm）
- 使用 `@page { size: A4; margin: 0; }` 控制打印区域
- **强约束：必须严格控制在 1 页 A4 以内。这是硬性要求，不可妥协。**
   - 应届生/初级岗位（工作经验 0-3 年）：绝对单页
   - 中高级岗位（5 年以上、内容确实丰富）：最多 2 页，但优先尝试压缩到 1 页
- 使用 `@media print` 优化打印效果
- 字体使用系统字体栈（中文: PingFang SC, Microsoft YaHei; 英文: 系统 serif/sans-serif 组合）

**设计约束**：
- 遵循内置美学标准（见 `references/design-guidelines.md`），精心选择字体组合、配色方案、空间排版
- **配色方案参考** `references/color-palettes.md`：风格-配色索引
- **CSS 按风格拆分** `references/css/`：每个风格独立 Markdown（3 套配色变量 + 字体 + 风格 CSS），通用排版权 `css/common.md`
- 拒绝使用 Inter、Roboto、Arial 等过度使用的字体
- 配色要有主色+点缀色，避免软弱的均匀调色板
- 关键信息（姓名、公司名、标题）要有层次感
- 使用 CSS 变量统一管理颜色
- 适当的 CSS 动画或微交互是可选的（但要克制）

**A4 布局要点**：
- 内容区域宽度控制在 190mm 以内（留 10mm 页边距）
- **紧凑排版参数（确保单页）**：
  - 上下页边距：≤ 12mm（最大化利用 A4 版面）
  - 正文行高：1.3-1.5（紧凑但不拥挤）
  - 板块间距：≤ 10mm（减少空白浪费）
  - 条目间距：≤ 4mm（列表项之间紧凑）
  - 姓名字号：22-26px；板块标题：13-15px；正文：10-11px
  - 技能标签使用语义化 `<ul>` 和 `<li>` 列表并通过 CSS inline-block 排列，确保 ATS 正确读取词汇
  - 教育背景可单行显示：学校 · 专业 · 时间
  - 使用 CSS `orphans` / `widows` 属性避免孤行
  - 优先双栏布局（左侧窄栏放技能/联系，右侧主栏放经历/项目）
- 板块之间有清晰的视觉分隔
- 打印时隐藏所有非必要元素（如按钮、背景色等）

**PDF 导出**：
- 在 HTML 中加入一个「导出 PDF」按钮
- 使用浏览器的 `window.print()` 实现打印为 PDF
- 打印时自动隐藏该按钮
- 添加一段说明文字提示用户：Chrome → 另存为 PDF

### 第三步半：PDF 导出验证（强制执行）

**⚠️ 此步骤不可跳过。生成 HTML 后必须立即执行。**

此步骤由实现 agent 独立执行。如果验证失败，agent 需要返回具体溢出情况（溢出行数/毫米数）及优化建议。收到反馈后，按需调整 CSS/内容，重新执行验证，重复直到通过。

1. **导出 PDF**：用无头浏览器将 HTML 导出为 PDF
   ```
   npx playwright pdf <html-file-url> <output-pdf-path>
   ```
   若 playwright 不可用，尝试：
   ```
   chrome --headless --disable-gpu --print-to-pdf=<output-pdf-path> --no-margins <html-file-url>
   ```

2. **检查页数**：
   ```
   python -c "import PyPDF2; r = PyPDF2.PdfReader('<pdf-path>'); print(f'页数: {len(r.pages)}')"
   ```

3. **判定标准**：
   - 页数 = 1 → ✅ 通过，继续交付
   - 页数 > 1 → ❌ 失败，必须修复后重试

4. **溢出修复策略**（按优先级顺序尝试）：
   1. 减小页边距（padding/margin → 最小可到 8mm）
   2. 压缩板块间距（gap/margin-bottom）
   3. 缩小行高（line-height: 1.25-1.35）
   4. 缩减字号（正文最低 9.5px，标题最低 12px）
   5. 精简文字描述（合并短句、删除冗余措辞）
   6. 切换为双栏布局（若当前为单栏）

### 第四步：交付

生成文件后，告诉用户：
1. 文件保存位置
2. 如何用浏览器打开和打印为 PDF
3. 下一步可以使用 jd-tailorer skill 针对具体 JD 进行定制

## 示例对话

**用户**: 帮我做一份简历
**Claude**: 好的，让我帮你构建一份精美的简历。首先，能告诉我你的姓名和求职意向吗？

**用户**: 我想换一个深色风格的简历
**Claude**: 明白，我会为你设计一个适合技术岗位的深色主题简历。信息我都收集好了，现在开始生成。

## 文件命名

- `resume.html` — 简历文件，保存在用户指定的位置
- 建议用户在项目根目录下保存，如 `resume/index.html`
- 导出 PDF 时建议命名：`姓名_岗位_学校_毕业时间.pdf`
- 投递 ATS 系统时：本 HTML 适合打印 PDF 和网页展示；如目标公司使用传统 ATS，建议同时提供 `.docx` 版本
